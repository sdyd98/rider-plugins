# Changelog — Log Viewer

이 파일은 Log Viewer 플러그인의 버전별 변경 사항을 기록합니다. (UI/릴리스 노트는 한국어, 코드/커밋은 영어.)

## [Unreleased]

### 추가
- **큰 로컬 파일은 꼬리부터 열기** — 100MB를 넘는 로컬 파일은 마지막 100MB만 읽어 수 초 만에 엽니다
  (원격의 `tail -n N`과 동일한 동작). 건너뛴 크기는 상태바에 "앞 2.9 GB 건너뜀"으로 표시됩니다.
- **로드 진행률 + 중지** — 초기 읽기 동안 상태바에 진행률(%)이 보이고, **중지** 버튼으로 로드를 끊고
  현재까지 읽은 내용으로 바로 볼 수 있습니다(라이브 tail은 파일 끝에서 이어짐).
- **SSH tail 자동 재연결** — 원격 tail 스트림이 끊기면(네트워크 순단, 서버 재시작) 이제 상태바에
  "원격 연결 끊김 — 재연결 중…"을 표시하고 지수 백오프(1초→최대 30초)로 자동 재연결합니다. 예전에는 IDE
  로그에만 남기고 화면은 라이브처럼 보였습니다. 재연결은 `tail -n 0 -F`를 다시 실행하므로 끊긴 동안의
  줄은 누락될 수 있습니다(중복 대신 누락 선택).
- **에러 히트맵(오버뷰 스트라이프)** — 그리드 오른쪽 가장자리에 파일 전체의 ERROR/WARN 위치를
  에디터 에러 스트라이프처럼 표시합니다(필터와 무관하게 전체 분포). 현재 화면 범위는 반투명 창으로
  보이고, 클릭/드래그로 해당 지점으로 점프합니다. `]e`/`[e` 레벨 점프도 레벨 인덱스를 이용해
  대용량에서 즉각 반응합니다.
- **파일별 형식/인코딩 기억 + 자동 감지** — 소스(로컬 경로 / `user@host:path`)별로 마지막에 쓴 줄
  형식과 인코딩을 기억해 다시 열 때 복원합니다. 처음 여는 파일은 저장된 형식들을 앞부분 줄에 대보고
  60% 이상 맞는 형식을 자동 적용합니다(⚙ → 줄 형식 라벨에 반영).

### 개선
- **기가바이트급 로그 로드 안정화** — 3GB(3,300만 줄) 파일 로드 중 IDE가 얼어붙던 문제를 고쳤습니다.
  힙 분석으로 확인한 원인 세 가지를 모두 제거:
  - **백프레셔 부재** — 리더 스레드가 파싱한 배치를 EDT가 소화하는 속도와 무관하게 무한정 큐잉해
    힙(2GB)이 가득 차 GC 스래싱으로 멈췄습니다(대기 배치 1,700개+, ParsedRow 350만 개 실측). 이제
    진행 중 배치를 4개로 제한하고 리더가 EDT를 기다립니다.
  - **로드 중 정렬러 유지비** — 필터가 걸린 `TableRowSorter`는 배치 삽입마다 O(전체 행) 인덱스
    재구축을 해서 EDT가 리더를 못 따라갔습니다. 이제 초기 스트리밍 동안 정렬러를 떼어 두고 로드
    완료(또는 첫 필터/접기 사용) 시점에 장착합니다 — xlsx 그리드와 같은 패턴.
  - **줄당 메모리 오버헤드** — `LogLine`의 `by lazy` 6개가 줄마다 래퍼+람다 12객체를 만들어 100만 줄
    상한 기준 ~380MB를 낭비했습니다. 필드 캐시로 교체.
- **20만+ 줄 로그 성능** — 대용량 로그에서 느려지던 세 병목을 제거했습니다.
  - **필터/검색이 블록 크기에 비례해 제곱으로 느려지던 문제** — 스택트레이스 같은 여러 줄 블록에서, 블록의
    각 줄마다 블록 전체를 다시 정규식 검사(k줄 블록이면 k²번)하고 블록 범위도 매번 선형 재탐색했습니다.
    이제 블록 크기를 O(1)로 캐시하고 필터 한 번에 각 줄을 정확히 한 번만 검사합니다 — 검색어 입력이
    스택트레이스 많은 20만+ 줄 로그에서도 즉각 반응합니다.
  - **접기(`za`) 토글마다 전체 재필터** — 이제 해당 블록의 행만 다시 필터링합니다(전체 모델 리빌드 없음).
  - **스트리밍 중 모든 줄의 픽셀 폭을 EDT에서 실측** — 컬럼 폭 계산이 로드 중 모든 줄을 FontMetrics로
    측정했습니다. 이제 "현재 최대 폭을 넘을 수 없는 줄"(길이×최대글리프폭 상한, CJK 2배폭 포함)은 건너뛰어
    사실상 새 최대 후보만 측정합니다.

## [0.4.1]

### 개선
- **비공백 구분 로그도 영역 지정 가능** — 영역 지정의 토큰을 이제 **공백뿐 아니라 구분자 `[ ] ( ) { } |` 로도
  쪼갭니다.** 그래서 `2026-09-01 09:00:01|ERROR|worker|메시지` 나 `[2026][ERROR][worker]메시지` 같은 공백으로
  안 나뉜 로그도 토큰을 클릭해 지정할 수 있습니다(타임스탬프 내부 `: - . /` 는 그대로 유지). 메시지는 시작 토큰부터
  끝까지 그대로입니다.

## [0.4.0]

### 추가
- **영역 지정(토큰 클릭)으로 형식 만들기** — 템플릿/정규식을 직접 안 쳐도, 줄 형식 창에서 **샘플 줄의 토큰을 클릭**해
  시간/레벨/스레드/메시지 영역을 지정하면 형식이 만들어집니다(⚙ → 줄 형식). **정규식은 노출하지 않습니다.**
  - 필드를 고르고 토큰을 클릭하면 **실제 로그 뷰어가 그 형식으로 바로 다시 나뉩니다**(별도 미리보기/만들기 버튼 없음).
  - **샘플 줄 ◀ ▶ 넘기기** — 맨 윗줄이 대표적이지 않을 수 있어, 여러 줄을 넘겨가며 대표 줄을 골라 지정할 수 있습니다.
  - **메시지는 시작 토큰 하나만** 클릭하면 그 뒤로 끝까지 자동 지정 — 긴 메시지를 일일이 안 눌러도 됩니다.
  - `[worker-3]` 같은 괄호 토큰은 괄호를 리터럴 처리하고 안쪽만 캡처. 지정 안 한 토큰은 와일드카드(`\S+`)라
    중간 필드를 건너뛰어도 다른 줄에 일반화됩니다.
  - **매칭된 형식은 잡은 것만** 보여줍니다(자동 분석과 병합 안 함) — 그래서 클릭할 때마다 컬럼이 눈에 띄게 채워집니다.
- **Thread 컬럼** — 형식에서 **스레드를 지정하면 뷰어에 Thread 컬럼이 생깁니다**(지정 안 하면 안 보임).
- **포맷 저장 + 드롭다운** — 만든 형식에 이름을 붙여 **저장**하고, **"적용 형식" 드롭다운에서 골라 적용**합니다.
  드롭다운의 **"없음 (원본 그대로)"** 를 고르거나 포맷을 다 지우면 컬럼 분리 없이 원본 줄로 보입니다.

### 변경
- **자동 분석(휴리스틱) 제거** — 이제 **적용된 포맷이 있을 때만** 시간/레벨/스레드/메시지로 나뉩니다. 포맷이 없거나
  형식에 안 맞는 줄은 **원본 그대로** 표시됩니다(예전엔 내장 추정으로 자동 분리). 포맷이 없으면 자동 스택 접기·레벨
  색·레벨 필터도 동작하지 않습니다 — 컬럼 기능을 쓰려면 영역 지정으로 포맷을 만들어 적용하세요.

### 수정
- **줄 형식 창 닫을 때 멈춤** — 형식 적용(재읽기)으로 그리드를 비운 직후 Swing이 사라진 행을 그리려다 발생하던
  `IndexOutOfBounds` 예외 폭주로 화면이 멈추던 문제 수정(`getValueAt`/`pushChrome`/`updateDetail`의 모델 행 가드).
- **실수로 바깥 클릭 시 닫힘 방지** — 줄 형식 창은 바깥을 눌러도 닫히지 않고, "닫기"/Esc로만 닫힙니다.

## [0.3.1]

### 줄 형식 UI 개선
- **미리보기 실시간 갱신** — 입력하는 즉시 미리보기가 갱신됩니다.
- **기본 템플릿 미리 입력** — 줄 형식 창을 처음 열면 `%{time} [%{thread}] %{level} %{message}`가 미리 채워져
  바로 편집할 수 있습니다.
- **안 맞는 줄은 원본으로** — 미리보기에서 형식이 매칭되지 않는 줄은 자동 분석 추정 대신 **원본 텍스트 그대로**
  표시해, 형식이 커버하지 못하는 줄을 한눈에 알 수 있습니다.

## [0.3.0]

### 추가
- **사용자 정의 줄 형식** — Time / Level / Message 컬럼을 나누는 기준을 직접 지정할 수 있습니다(⚙ → "줄 형식").
  - **템플릿** `%{time} [%{thread}] (%{level}) %{message}` 또는 **named-capture 정규식** `(?<time>…)(?<level>…)(?<message>…)`
    입력. 템플릿은 리터럴(`[ ] ( )` 등)을 그대로 매칭하고 공백은 유연하게 처리.
  - 규칙은 **순서 목록** — 위에서부터 시도하고, 안 맞으면 **기존 자동 분석(휴리스틱)으로 폴백**(빈 목록이면 예전과 동일).
  - **Compose/Jewel 설정 + 실시간 미리보기** — 현재 로그 앞부분이 입력에 맞춰 Time/Level/Message로 즉시 쪼개져 보임.
  - 닫으면 적용(변경 시 재읽기). 규칙은 영속화(`LineFormatStore`).

### 수정
- **디테일 패널 갱신 안 됨** — 행을 위아래로 이동할 때 이전 줄의 상세가 남고 갱신되지 않던 문제 수정. 패널에
  연속 애니메이션이 없어 Compose 프레임 루프가 idle일 때 EDT의 상태 변경을 놓치던 것 → 스냅샷 플러시 + repaint로 해결.

## [0.2.2]

### 수정 / 성능
- **대용량 로그 + 인코딩 변경 시 멈춤** — 줄당 파싱(정규식 5개 + ANSI 처리)을 **백그라운드 스레드로 이동**.
  이전엔 UI 스레드에서 파싱해, 5만 줄짜리 로그를 인코딩 변경으로 다시 읽으면 그리드가 멈췄습니다. 이제 리더
  스레드가 파싱하고 UI 스레드는 삽입만 하므로, 대용량 **초기 로드와 인코딩 재읽기 모두** 화면이 반응성을 유지합니다.
  (0.2.1의 백그라운드 close 수정에 더해, 파싱까지 EDT에서 들어냈습니다.)

### 인코딩 UI
- **인코딩 피커를 Compose/Jewel로 재작성** — 기존 스윙 리스트(`createPopupChooserBuilder`)를 걷어내고, 나머지
  크롬과 톤을 맞춘 Compose 팝업(현재 인코딩 체크 ✓ + hover)으로 교체.
- **목록을 UTF-8 + 한글만으로 정리** — `UTF-8` / `CP949 · Windows-949 (한글 · ANSI)` / `EUC-KR`. ISO-8859-1·US-ASCII
  제거. CP949 항목이 곧 한글 Windows의 "ANSI"(`native.encoding` = MS949)라 별도 ANSI 항목은 두지 않음.

## [0.2.1]

### 수정
- **인코딩 변경 시 화면 멈춤** — 활성 로그 뷰어에서 인코딩을 바꾸면 화면이 멈추던 문제 수정. 이전 리더를
  EDT(UI 스레드)에서 닫느라 원격(SSH) 연결 끊기의 네트워크 I/O가 UI를 막았습니다. 이제 이전 리더는 백그라운드
  스레드에서 닫고, 재읽기 도중 이전 리더가 흘리는 배치는 무시(generation 가드)해 새로 비운 그리드에 섞이지
  않습니다.

## [0.2.0]

### 추가
- **인코딩 선택** — ⚙ 표시 옵션의 **인코딩** 항목에서 디코딩 인코딩을 **UTF-8 / CP949(한글 Windows) / EUC-KR /
  ISO-8859-1 / US-ASCII** 중 고릅니다. 선택하면 소스를 그 인코딩으로 **즉시 다시 읽습니다**. CP949/EUC-KR로 저장된
  한글 로그가 깨져 보일 때 바로 고칠 수 있고, 잘못 골랐어도 다시 바꿀 수 있습니다. (줄 분리는 바이트 단위 `0x0A`
  스캔이라 UTF-16/UTF-32처럼 `0x0A`를 내부 바이트로 쓰는 인코딩은 의도적으로 제외했습니다.)
- **에디터 폰트 적용** — 로그 본문 폰트가 Rider **에디터 폰트**(Settings → Editor → Font)를 따라갑니다(이전엔 논리
  Monospaced 고정). 한글/CJK 글리프가 좋은 폰트를 쓰면 로그에도 그대로 적용됩니다. (Settings 변경은 다음 IDE 재시작에
  반영.)

## [0.1.0] — 첫 릴리스

### 추가
- **로컬 & 원격(SSH/SFTP) 로그 뷰어** — 로컬 `.log` 파일은 에디터 탭으로, 원격 호스트는 **로그 뷰어** 툴윈도우에서
  Rider의 SSH 설정을 골라 연결해 봅니다. 연결은 루트 `/`에서 시작하고, 폴더를 우클릭해 로그 루트로 고정합니다.
  (`.log` 파일과 그것을 포함한 폴더만 트리에 보입니다.)
- **라이브 follow** — 로컬은 폴링 바이트 커서, 원격은 `tail -F`. Follow 토글로 하단 고정(스크롤을 올리면 자동 일시정지).
- **필터 · 검색 · 하이라이트** — 전체 줄 정규식 필터, 레벨별 토글(ERROR/WARN/INFO/DEBUG/TRACE, 라이브 카운트),
  GrepConsole 스타일 색상 규칙(정규식 + 색/볼드, 다이얼로그로 편집·영속), 검색 매치 인플레이스 하이라이트.
- **구조 분석** — 스택트레이스/들여쓰기 블록 접기, `J` JSON 정렬 팝업, `T` 시각으로 이동.
- **3열 레이아웃**(Time · Level · Message) + ANSI(SGR) 색상 렌더링, ⚙ 메뉴의 원본 로그 보기 전환.
- **항상 켜진 vim 내비게이션** — hjkl(컬럼/메시지 안 문자), w/e/b 단어 모션, v/V 비주얼 선택·복사(y/yy),
  gg/G, Ctrl+D/U, zz, n/N, `*`/`#`, gt/gT(파일 탭), `]e`/`[e` 레벨 점프, 마크, `?` 도움말. 트리도 vim으로 이동.

### 메모
- 읽기 전용 — 파일이나 원격 호스트에 아무것도 쓰지 않습니다. SSH 비밀번호는 IDE의 PasswordSafe에만 저장되고
  직렬화된 프로필에는 남지 않습니다. Host-key 검사는 비활성(개발 도구, 직접 관리하는 머신 대상).
- SSH는 JSch(mwiede 포크)를 플러그인 `lib/`에 번들. Compose/Jewel은 IDE가 제공하므로 ZIP에 포함되지 않습니다.
