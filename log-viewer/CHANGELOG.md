# Changelog — Log Viewer

이 파일은 Log Viewer 플러그인의 버전별 변경 사항을 기록합니다. (UI/릴리스 노트는 한국어, 코드/커밋은 영어.)

## [0.2.2]

### 수정 / 성능
- **대용량 로그 + 인코딩 변경 시 멈춤** — 줄당 파싱(정규식 5개 + ANSI 처리)을 **백그라운드 스레드로 이동**.
  이전엔 UI 스레드에서 파싱해, 5만 줄짜리 로그를 인코딩 변경으로 다시 읽으면 그리드가 멈췄습니다. 이제 리더
  스레드가 파싱하고 UI 스레드는 삽입만 하므로, 대용량 **초기 로드와 인코딩 재읽기 모두** 화면이 반응성을 유지합니다.
  (0.2.1의 백그라운드 close 수정에 더해, 파싱까지 EDT에서 들어냈습니다.)

### 인코딩 UI
- **인코딩 피커를 Compose/Jewel로 재작성** — 기존 스윙 리스트(`createPopupChooserBuilder`)를 걷어내고, 나머지
  크롬과 톤을 맞춘 Compose 팝업(현재 인코딩 체크 ✓ + hover)으로 교체.
- **목록을 UTF-8 + 한글만으로 정리** — `UTF-8` / `CP949 · Windows-949 (한글 · ANSI)` / `EUC-KR`. ISO-8859-1·US-ASCII
  제거. CP949 항목이 곧 한글 Windows의 "ANSI"(`native.encoding` = MS949)라 별도 ANSI 항목은 두지 않음.

## [0.2.1]

### 수정
- **인코딩 변경 시 화면 멈춤** — 활성 로그 뷰어에서 인코딩을 바꾸면 화면이 멈추던 문제 수정. 이전 리더를
  EDT(UI 스레드)에서 닫느라 원격(SSH) 연결 끊기의 네트워크 I/O가 UI를 막았습니다. 이제 이전 리더는 백그라운드
  스레드에서 닫고, 재읽기 도중 이전 리더가 흘리는 배치는 무시(generation 가드)해 새로 비운 그리드에 섞이지
  않습니다.

## [0.2.0]

### 추가
- **인코딩 선택** — ⚙ 표시 옵션의 **인코딩** 항목에서 디코딩 인코딩을 **UTF-8 / CP949(한글 Windows) / EUC-KR /
  ISO-8859-1 / US-ASCII** 중 고릅니다. 선택하면 소스를 그 인코딩으로 **즉시 다시 읽습니다**. CP949/EUC-KR로 저장된
  한글 로그가 깨져 보일 때 바로 고칠 수 있고, 잘못 골랐어도 다시 바꿀 수 있습니다. (줄 분리는 바이트 단위 `0x0A`
  스캔이라 UTF-16/UTF-32처럼 `0x0A`를 내부 바이트로 쓰는 인코딩은 의도적으로 제외했습니다.)
- **에디터 폰트 적용** — 로그 본문 폰트가 Rider **에디터 폰트**(Settings → Editor → Font)를 따라갑니다(이전엔 논리
  Monospaced 고정). 한글/CJK 글리프가 좋은 폰트를 쓰면 로그에도 그대로 적용됩니다. (Settings 변경은 다음 IDE 재시작에
  반영.)

## [0.1.0] — 첫 릴리스

### 추가
- **로컬 & 원격(SSH/SFTP) 로그 뷰어** — 로컬 `.log` 파일은 에디터 탭으로, 원격 호스트는 **로그 뷰어** 툴윈도우에서
  Rider의 SSH 설정을 골라 연결해 봅니다. 연결은 루트 `/`에서 시작하고, 폴더를 우클릭해 로그 루트로 고정합니다.
  (`.log` 파일과 그것을 포함한 폴더만 트리에 보입니다.)
- **라이브 follow** — 로컬은 폴링 바이트 커서, 원격은 `tail -F`. Follow 토글로 하단 고정(스크롤을 올리면 자동 일시정지).
- **필터 · 검색 · 하이라이트** — 전체 줄 정규식 필터, 레벨별 토글(ERROR/WARN/INFO/DEBUG/TRACE, 라이브 카운트),
  GrepConsole 스타일 색상 규칙(정규식 + 색/볼드, 다이얼로그로 편집·영속), 검색 매치 인플레이스 하이라이트.
- **구조 분석** — 스택트레이스/들여쓰기 블록 접기, `J` JSON 정렬 팝업, `T` 시각으로 이동.
- **3열 레이아웃**(Time · Level · Message) + ANSI(SGR) 색상 렌더링, ⚙ 메뉴의 원본 로그 보기 전환.
- **항상 켜진 vim 내비게이션** — hjkl(컬럼/메시지 안 문자), w/e/b 단어 모션, v/V 비주얼 선택·복사(y/yy),
  gg/G, Ctrl+D/U, zz, n/N, `*`/`#`, gt/gT(파일 탭), `]e`/`[e` 레벨 점프, 마크, `?` 도움말. 트리도 vim으로 이동.

### 메모
- 읽기 전용 — 파일이나 원격 호스트에 아무것도 쓰지 않습니다. SSH 비밀번호는 IDE의 PasswordSafe에만 저장되고
  직렬화된 프로필에는 남지 않습니다. Host-key 검사는 비활성(개발 도구, 직접 관리하는 머신 대상).
- SSH는 JSch(mwiede 포크)를 플러그인 `lib/`에 번들. Compose/Jewel은 IDE가 제공하므로 ZIP에 포함되지 않습니다.
